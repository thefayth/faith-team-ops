# Project Brief

## Name

Faith Team Ops

## Short Description

Faith Team Ops is a private WordPress-based operations platform for distributed teams, designed to support check-ins, reports, support requests, scoped dashboards, private media, audit receipts, and role-aware coordination.

## Purpose

The project exists to turn scattered operational coordination into a private, structured, auditable system of record. It is built for real teams that need mobile-first workflows, clear ownership, careful access boundaries, and receipts after important actions.

## Audience

- Faith, as owner and strategic operator.
- Admins managing global operations.
- Country leads coordinating local work.
- Contractors and volunteers submitting work, reports, and support needs.
- Moderators or observers who need limited review access.

## Public Positioning

Faith Team Ops is not a generic dashboard. It is a private operating room for serious distributed work, with public materials that explain the concept while protecting the engine.

## Core Workflow

1. A team member enters through the WordPress portal.
2. Their role and country scope determine available actions.
3. They submit check-ins, reports, tasks, support requests, or private media metadata.
4. Country leads and admins review the work.
5. The system preserves audit receipts and follow-up status.

## Visual Direction

The current public package includes three implementation directions:

- Signal Desk: safest daily-ops dashboard.
- Field Command: country-ops command center.
- Care Ledger: care-forward support and accountability ledger.

These directions are shown in the public visual board and WordPress draft.
