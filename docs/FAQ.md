# FAQ

## Is Faith Team Ops open source?

No. This is a protected public project surface. No source release is made or implied.

## Can I use the code?

No. The source code is not included, and no public license is granted.

## Can I use the visuals or diagrams?

No. The visuals and diagrams are protected project assets. They may be viewed as part of this public project surface but are not reusable without written permission.

## What does the public repository show?

It shows the project purpose, status, workflow model, visual directions, public/private boundary, ownership posture, and WordPress page draft.

## What remains private?

The private WordPress plugin source, deployment infrastructure, credentials, operational records, private media, internal prompts, agent instructions, and private workflows remain excluded.

## Which visual direction is selected?

No single direction is selected for the private portal UI in this public package. The repository intentionally shows all three current directions: Signal Desk, Field Command, and Care Ledger.

## Is the platform live?

This public package does not claim a full production launch. It reflects a public surface candidate and documents that private launch gates remain separate.

## How do I learn more?

Start with `README.md`, `docs/PROJECT_BRIEF.md`, `docs/WORKFLOW_DIAGRAMS.md`, and the WordPress draft in `wordpress/page.md`.
